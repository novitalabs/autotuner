### Issue Analysis

GitHub Issue #3 reported that `analyze_slo_violations()` fails with:
```
AttributeError: 'Task' object has no attribute 'task_config'
```

### Root Cause

In `src/web/agent/tools/task_tools.py` line 967:
```python
slo_config = task.task_config.get("slo", {})
```

The `Task` model (defined in `src/web/db/models.py`) does not have a `task_config` attribute. The correct attribute for SLO configuration is `slo_config` (line 42 in models.py).

### Fix Applied

Changed line 967 from:
```python
slo_config = task.task_config.get("slo", {})
```

To:
```python
slo_config = task.slo_config or {}
```

This correctly accesses the `slo_config` JSON column defined in the Task model.
